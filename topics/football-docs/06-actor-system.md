# ACT — 人物系统

> 目标：球员不是能力值容器，而是带着欲望、压力和记忆上场的人。
> 人物系统是「架空球员情感冷启动」风险的第一道答案：让玩家在 2-3 小时内
> 对至少 3 个虚构人物产生情绪。

### ACT-01 人格模型与特质

状态: draft · 优先级: P0 · 依赖: 无

**目的**：给每个 Actor 一组少而可读、机制相关的人格参数，作为一切行为
判定（ACT-05）、事件前置（EVT-01）、文本语气（EVT-06）的根。

**设计理念**：产品红线「Traits 要少、可读、机制相关」。维度负责连续
差异，特质负责离散戏剧性；两层都必须在机制里有真实作用，禁止装饰性
人格。

**如何设计**：
- 6 维人格（0-100）：野心 / 忠诚 / 职业心 / 情绪稳定 / 社交性 / 自尊心；
  每维在 ≥ 3 个引擎中有消费点（注明在引擎卡片中）；
- 特质标签池 12-18 个，每个 = {触发场景, 机制效果, 文本语气键}，如：
  火爆（冲突候选 ×2，红牌率 ×1.5）、玻璃心（批评的心情伤害 ×2，表扬
  增益 ×2）、大场面（关键战发挥 +，MAT-04）、隐形人（媒体曝光 ×0.5，
  焦点权重 ×0.5）、更衣室领袖（派系凝聚贡献 ×2）、记仇（负面记忆衰减
  ×0.5）、老好人（和解候选 ×2）；每 Actor 持有 1-3 个；
- 人格生成：世界生成时按地区/文化标签调制的分布抽样（WV-01/03），
  禁止全队同质化（队内每维方差下限）；
- 人格基本不变：人格是常量，状态（ACT-02）才是变量；仅 season_defining
  级事件可永久微调 ±5 并记日志（创伤/蜕变，稀缺性保证其叙事重量）。

**验收标准**：
- [机器] 每个维度/特质在代码中有 ≥ 3 / ≥ 1 处真实消费点（引用扫描）；
- [机器] 生成的球队人格维度方差 ≥ 下限；特质覆盖率：池中每特质在
  300 Actor 中出现 3-40 次；
- [LLM] 给评估器某 Actor 的赛季事件流（隐藏人格卡），反推人格与实际
  卡的匹配度 ≥ 70%——人格必须能从行为中被读出来。

**评估钩子**：
- 特质的事件参与率（某特质长期不进事件 = 装饰性人格，违规）。

### ACT-02 状态模型（情绪 / 压力 / 自信）

状态: draft · 优先级: P0 · 依赖: ACT-01, CAT-02

**目的**：承载人物的快变量，把外界刺激转化为表现波动与行为倾向，
是「人心不稳定」（G1）的数值载体。

**设计理念**：状态是刺激的积分器，带人格调制的衰减回归：同样输球，
情绪稳定 80 的老将三天平复，玻璃心新星一周低迷并积累压力。

**如何设计**：
- 三个核心状态量（0-100）：心情（短期，半衰期 3-7 日按情绪稳定调制）、
  压力（中期积分器，来源：出场失意/媒体负面/老板点名/家庭标签/合同年）、
  自信（表现反馈回路：连续好评分↑、被弃用/失误↓）；
- 状态→表现：比赛发挥系数 = f(心情, 自信, 压力)，压力呈倒 U 曲线
  （适度压力提升，过载崩盘——大场面特质改变曲线形状，MAT-03 消费）；
- 状态→行为：低心情提高冲突/抱怨候选权重，高压力解锁压力释放事件
  （DIR-02 个体压力线），低自信提高逃避型判定（ACT-05）；
- 可见性设计（G1）：玩家看到的是模糊化档位（很好/不错/低落/糟糕）+
  助教报告（带助教能力噪声），精确数值永不直接展示——不确定性是
  特性。

**验收标准**：
- [机器] 状态值始终在界内；衰减回归半衰期与人格调制公式单测通过；
- [机器] 压力倒 U 曲线：构造高/中/低压力同能力球员各 1000 场模拟，
  发挥均值中压组最高；
- [LLM] 抽 5 名球员赛季状态轨迹+事件流，「状态变化是否可解释」≥ 4/5。

**评估钩子**：
- 全联赛状态分布的赛季动态（开季乐观→中期分化→收官极化为健康形态）。

### ACT-03 记忆系统

状态: draft · 优先级: P0 · 依赖: ACT-01, ENG-04

**目的**：让世界有记性。记忆是事件链（EVT-04）、长期恩怨（DIR-05）、
人物深度感的存储基础——「会记住你的球员」是商店页第一卖点。

**设计理念**：记忆 ≠ 日志。日志是世界的客观记录，记忆是 Actor 的主观
摘录：同一事件不同人记住的版本不同、显著度不同、衰减不同。

**如何设计**：
- 记忆条目：`{owner, 类型, 源事件 cause_id, 主观极性(-100~+100),
  显著度(0-100), 写入 tick, 涉及他人[], 主观摘要键}`；
- 记忆类型 10 种（MVP）：被承诺 / 承诺被弃 / 被信任重用 / 被冷落 /
  被公开批评 / 被保护 / 关键战立功 / 关键战罪人 / 受伤经历 / 恩怨事件；
- 主观化规则：写入时按人格调制——记仇特质负面显著度 ×1.5，自尊心高
  者对公开批评极性加深；同一事件可给多人写不同记忆（带伤上场：本人
  记「被牺牲/被信任」分支，队友记「教练会牺牲人」，CAT-02）；
- 衰减与固化：显著度每月 -5（记仇减半）；被事件引用一次显著度 +15
  （越提越记得牢）；显著度 < 10 转入归档（可检索不参与候选）；
- 容量：活跃记忆每 Actor ≤ 40（ENG-06），溢出挤掉最低显著度；
- 消费接口：事件前置谓词（EVT-01）、关系修正（REL-01 读取双方相关
  记忆）、文本渲染槽（EVT-06 引用记忆摘要）、玩家可在人物页查看
  （UX-03——玩家能看见自己留下的伤痕）。

**验收标准**：
- [机器] 每条记忆的 cause_id 可回溯；归档/挤出零数据丢失；
- [机器] 跑批中记忆引用事件占比 ≥ 15%（EVT-04 同口径）；
- [LLM] 抽 10 个「旧记忆被引用」时刻，「呼应自然度」≥ 4/5；
- [人工] 人物页记忆列表读起来像小传而非日志转储。

**评估钩子**：
- 记忆类型分布；平均存活期；引用次数 top 记忆的内容审查（最常被讲的
  故事是什么）。

### ACT-04 欲望与个人目标

状态: draft · 优先级: P1 · 依赖: ACT-01, ACT-02

**目的**：给每个 Actor 一个「他自己想要什么」，使其行为有内在一致的
方向感，而不是纯被动反应器——人物有戏的前提是有所求。

**设计理念**：欲望产生期望，期望落差产生压力与事件。玩家管理的本质
是管理一屋子互相冲突的欲望（G1）。

**如何设计**：
- 欲望槽（每 Actor 1-2 个主导欲望，由人格+生涯阶段生成）：出场时间 /
  冠军荣誉 / 加薪合同 / 转会豪门 / 队长袖标 / 主力地位 / 善终退役 /
  复仇（特定对象）/ 被偶像认可；
- 满足度账本：每周结算欲望满足度（如出场欲望 vs 实际出场分钟），
  持续落差转化为压力（ACT-02）与不满候选（EVT-02）；
- 欲望可见性：玩家通过私下谈话（UX-02）或助教报告获知，带噪声；
  承诺机制（「我保证给你机会」）写入被承诺记忆（ACT-03），兑现与否
  是信任分水岭；
- 欲望演化：生涯阶段切换（ACT-06）或 season_defining 事件可更换主导
  欲望（夺冠后野心→善终，被背叛后忠诚→复仇）。

**验收标准**：
- [机器] 每 Actor 恒有 ≥ 1 个主导欲望；满足度结算无遗漏；
- [机器] 欲望落差 top10% 的 Actor，其不满类事件触发率显著高于均值；
- [LLM] 「该 Actor 的赛季行为与其欲望是否一致」≥ 4/5。

**评估钩子**：
- 欲望类型分布；承诺兑现率 vs 信任变化的相关性（应强正相关）。

### ACT-05 行为判定接口

状态: draft · 优先级: P0 · 依赖: ACT-01, ACT-02, ACT-04, REL-01

**目的**：统一回答「此人此情境会怎么做」。所有引擎需要人物做选择时
（服从战术？接受替补？怎么回应批评？）调用同一判定，保证同一个人在
不同系统里是同一个人。

**设计理念**：判定 = 人格基线 + 状态修正 + 关系修正 + 欲望利害 +
确定性随机。集中实现，杜绝各引擎私设人格逻辑导致人设分裂。

**如何设计**：
- 接口：`decide(actor, situation, options[]) -> (choice, rationale)`；
  situation 是带标签的情境描述（被替补/被批评/队友被欺负/记者套话）；
- 评分函数：每个 option 按四组系数打分——人格倾向表（每种情境预定义
  6 维权重）、当前状态偏置、对在场他人的关系偏置、欲望利害（该选择
  推进/损害主导欲望的程度）；softmax 温度由情绪稳定调制（不稳定的人
  更随机）；
- rationale 必须返回结构化理由（哪一项主导了选择），写入日志——
  「他为什么这么做」永远可查（G4），也是 LLM 评估人设一致性的输入；
- MVP 情境表 12-16 种，覆盖训练/比赛/更衣室/媒体四场景。

**验收标准**：
- [机器] 同 Actor 同情境同状态判定结果确定（G5）；rationale 字段无空；
- [机器] 构造极端人格 Actor（全 95 vs 全 5）跑同批情境，选择分布差异
  显著；
- [LLM] 人设一致性盲测：给 20 个判定记录（隐藏人格），反推人格命中率
  ≥ 70%（与 ACT-01 同口径）。

**评估钩子**：
- rationale 主导因素分布（长期被单一因素主导 = 权重失衡）。

### ACT-06 成长、衰退与生涯阶段

状态: draft · 优先级: P1 · 依赖: ACT-01, CAT-01

**目的**：给人物时间维度：天才出场、巅峰、老去、退场。多赛季世界线
（DIR-05）的微观基础。

**设计理念**：年龄曲线是结构性慢变量；衰退期的尊严处理（老将让位、
转型、告别战）是高质量叙事素材库，机制必须为其留好接口。

**如何设计**：
- 生涯阶段：新秀(≤20) / 上升(21-24) / 巅峰(25-29) / 黄昏(30-33) /
  谢幕(34+)，界限受职业心与伤病史调制；
- 成长率曲线：新秀高方差（妖星与水货并存）、巅峰平台、黄昏负增长
  （身体技能先衰，意识类技能缓衰甚至微涨）；
- 阶段事件钩子：新秀首秀 / 突破赛季 / 巅峰大合同 / 黄昏让位冲突 /
  谢幕告别（每阶段 ≥ 3 个专属事件定义，入 EVT-07 池）；
- 退役判定与转型：谢幕期综合（能力/出场/欲望）判定退役，转型概率表
  → 教练/经纪人/评论员/离开足球（DIR-05 消费）。

**验收标准**：
- [机器] 10 赛季跑批：联赛年龄结构稳态（无全联盟老龄化/婴儿化漂移）；
- [机器] 每赛季退役 Actor 数在区间，转型者次季参与 ≥ 1 事件；
- [LLM] 抽老将生涯末两季事件流，「退场弧线完整度」≥ 3.5/5。

**评估钩子**：
- 能力-年龄散点的赛季演化；新秀成材率分布。

### ACT-07 非球员 Actor

状态: draft · 优先级: P1 · 依赖: ACT-01, WV-04

**目的**：老板、记者、助教、经纪人、家人以多大颗粒度进入 Actor 系统
——全量模拟做不起（G8），全静态没有戏。

**设计理念**：分级建模：参与日常判定的做全 Actor，只提供影响的做
标签/参数包。

**如何设计**：
- 全 Actor（有人格/状态/记忆/关系）：老板、记者、助理教练、玩家主
  教练本人、对手主教练（NPC 教练，DIR-05）；
- 半 Actor（有立场账本无逐日状态）：球迷组织（CAT-05）、协会（WV-04）；
- 标签化（挂在球员身上的修正器）：家人朋友（家庭压力/狐朋狗友/模范
  家庭 等 4-6 种标签，影响压力来源与事件前置，ACT-02/EVT-01）；
- 经纪人 MVP 处理：半 Actor，立场=利益最大化，仅在转会窗与续约事件
  中激活 [正式版升级全 Actor 待评估]；
- 助理教练特殊职责：玩家的信息过滤器——状态报告、欲望侦察、更衣室
  风声都经他转述，其能力与偏见决定情报噪声（G1 不确定性的一环）。

**验收标准**：
- [机器] 全 Actor 数 ≤ 60（球员之外），内存与 tick 耗时在 ENG-06 预算内；
- [机器] 每类非球员 Actor 至少出现在 3 种事件定义中；
- [LLM] 助教报告抽样「有用但不全知」的体感评分 ≥ 3.5/5。

**评估钩子**：
- 非球员 Actor 的事件参与占比（目标 25-40%，过低=世界只有球员）。
