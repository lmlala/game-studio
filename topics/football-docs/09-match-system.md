# MAT — 比赛系统

> 比赛是叙事的高压锅：人物状态、关系、战术在 90 分钟内碰撞结算。
> 双重约束：结果必须统计可信（G2），过程必须叙事可用（G3/G4）。
> 本系统是文本/关键时刻模拟，与 2D demo（rust/src/engine/）解耦。

### MAT-01 比赛模拟抽象层级

状态: draft · 优先级: P0 · 依赖: ENG-03, ENG-06

**目的**：选定比赛模拟的颗粒度：足够细以产生有因果的关键时刻，
足够粗以保证整赛季秒级模拟与可调试性（G8）。

**设计理念**：不做物理模拟（2D demo 另算），做**回合制控球段模型**：
比赛 = 一串控球段（possession），每段经过推进-机会-终结判定链。
这是 OOTP 式统计模拟与叙事钩子的折中。

**如何设计**：
- 比赛 = 90 分钟 ≈ 80-110 个控球段 [待评估校准]；每段：
  控球方推进判定（中场质量+节奏战术）→ 机会创造判定（创造力+
  对方防守组织）→ 终结判定（射手能力+门将+状态系数）；
- 每个判定消费具体球员的（技能 × 状态 ACT-02 × 关系修正 REL-05 ×
  战术角色 MAT-02），并记录参与者——进球链路上每个人有名有姓；
- 时间结构：上下半场 + 补时；体能随分钟衰减影响判定（CAT-02 联动）；
  换人窗口 3 次（玩家命令或 AI 教练判定）；
- 非控球事件层：犯规/红黄牌/伤病/冲突在段间按状态与人格抽样
  （火爆特质↑红牌，ACT-01 消费点）；
- 输出：比分、关键时刻序列（MAT-05）、个人评分、统计行（射门/
  控球率/犯规——支撑媒体与玩家复盘）。

**验收标准**：
- [机器] 单场模拟 < 50ms；整赛季全部比赛 < 5s（ENG-06 内）；
- [机器] 每个进球的判定链完整可回溯（谁推进/谁创造/谁终结/谁失位）；
- [机器] 控球段数、射门数分布在现实带（场均射门 8-18 次/队）。

**评估钩子**：
- 判定链各环节的贡献度分布（某环节长期不影响结果 = 死参数）。

### MAT-02 战术模型

状态: draft · 优先级: P0 · 依赖: MAT-01, ACT-05

**目的**：给玩家有限但有真实杠杆的赛前控制面（G1）：阵型、心态、
节奏、压迫、核心角色——并让执行度受人心调制。

**设计理念**：战术影响「分布」而不是「剧本」；执行噪声来自球员
（信任/人格/状态），玩家学到的是「这套人马吃不吃这套打法」。

**如何设计**：
- 五项设置（继承现有 TacticalSettings 概念并接通）：
  阵型（4-5 种模板：442/433/352/451/532）→ 位置覆盖与对位矩阵；
  心态（5 档防守-进攻）→ 控球段风险偏好；
  节奏（3 档）→ 控球段数与体能消耗；
  压迫（3 档）→ 夺回率与犯规/体能代价；
  核心角色（组织核/防守核/进攻核各指定 1 人）→ 该球员判定参与
  权重 ×1.5 且承担更多评分波动（捧人/毁人机制，叙事钩子）；
- 执行度 = f(平均战术理解, 对教练信任 REL-05, 训练磨合周数)；
  执行度低时实际参数向球员习惯偏移并产生「无视战术」match event
  （ACT-05 判定，谁的锅有名有姓）；
- 对手 AI 教练：NPC 教练按其风格模板（DIR-05 的 NPC 教练人设）
  选战术并做中场调整——玩家的博弈对象是有人格的教练而非随机表；
- 克制关系温和化：阵型/风格相性修正 ≤ ±10%（拒绝石头剪刀布化，
  G2 可信度）。

**验收标准**：
- [机器] 五项设置各自独立可测：单变量扫描跑批，对应统计量单调或
  按设计响应（如节奏↑→控球段数↑→体能消耗↑）；
- [机器] 执行度噪声有界；「无视战术」事件率与信任值负相关显著；
- [LLM] 战术复盘文本「玩家能否理解为什么打不出来」≥ 4/5。

**评估钩子**：
- 各阵型/心态组合的胜率矩阵（防出现必胜套路：任意组合胜率差
  ≤ 15%）。

### MAT-03 个人发挥判定

状态: draft · 优先级: P0 · 依赖: ACT-02, REL-05, MAT-01

**目的**：回答「他今天踢得怎么样、为什么」。个人发挥是赛后叙事
（英雄/罪人）与玩家用人决策反馈的核心。

**设计理念**：发挥 = 能力基线 + 状态系数 + 压力响应 + 关系化学 +
比赛内动量；方差按人格调制——稳定老将与过山车天才的区别是方差
不是均值（人格的比赛表达）。

**如何设计**：
- 发挥系数 = 技能 ×（心情/自信复合 0.85~1.15）×（压力倒 U，
  ACT-02）×（默契修正 ±8%，REL-05）×（场合系数：德比/关键战放大
  压力项，大场面特质反转符号，ACT-01）；
- 比赛内动量：早段表现影响后段判定（进球涨自信、早失误高压者
  可能滚雪球崩盘——崩盘链是合法戏剧，但单场限幅 ±0.1）；
- 评分映射：判定链参与度 + 正负事件加权 → 1-10 评分（媒体与球迷
  引擎消费，CAT-03/05）；
- 隐藏信息（G1）：玩家赛前只见档位化状态与助教建议；「他今天能不能
  顶住」的不确定性是首发决策的张力来源。

**验收标准**：
- [机器] 同能力不同人格的球员长期评分：均值差 < 0.3、方差差显著
  （人格→方差的设计验证）；
- [机器] 评分分布全联赛均值 6.5±0.3、单场极值 ≤ 10≥ 1；
- [LLM] 抽 10 个高/低分表现的赛后叙述「发挥可解释」≥ 4/5。

**评估钩子**：
- 压力响应曲线的实测形状；大场面/玻璃心特质球员在关键战的评分
  分布对比（应显著分化）。

### MAT-04 关键时刻引擎

状态: draft · 优先级: P1 · 依赖: MAT-01, MAT-03, DIR-07

**目的**：从控球段流水中识别并强化「值得讲的时刻」：绝杀、失点、
门线救险、染红离场、宿敌对决——战报与比赛文本直播的素材源。

**设计理念**：关键时刻 = 客观重要性（比分/时间敏感）× 主观叙事
价值（参与者的弧线/记忆/关系背景）。同一脚射门，宿敌德比补时阶段
由被弃用老将完成——价值完全不同。Director 不改变判定结果，只把
镜头对准它（DIR-01 纪律：不创造事实）。

**如何设计**：
- 客观分：比赛时间（补时×3）、比分差（平局/一球差×2）、赛季利害
  （保级/争冠战×2）；
- 叙事分：参与者的活跃弧线（DIR-06）、相关记忆（旧主对决/承诺
  兑现场景）、关系背景（怨恨对位）——从世界状态查询加权；
- 综合分超阈的时刻升格为 key moment：完整因果展开（参与者视角、
  背景引用）、写入高显著度记忆（ACT-03）、进入战报候选与媒体选题
  （CAT-03）；
- 每场 key moment 数量 2-6 个 [待评估校准]，超出取 top（镜头经济，
  DIR-07 同理念）。

**验收标准**：
- [机器] key moment 100% 带客观分+叙事分明细；数量在带内；
- [机器] 含记忆/弧线引用的 key moment 占比 ≥ 30%（镜头确实在跟
  故事走）；
- [LLM] 抽 10 个 key moment 文本「这球值得记住」≥ 4/5。

**评估钩子**：
- key moment 的记忆写入→后续引用转化率（关键时刻是否真的成为
  长期叙事素材）。

### MAT-05 比赛事件输出契约

状态: draft · 优先级: P0 · 依赖: MAT-01, ENG-04

**目的**：定义比赛产出的结构化事件流，供文本直播、赛后事件候选、
媒体引擎、评估管线四方消费——比赛与叙事世界的对接面。

**设计理念**：比赛引擎对外只说结构化语言；任何文本表现（直播/
战报）都是下游渲染（G4）。

**如何设计**：
- match event 类型表：goal / chance / save / foul / card / injury /
  sub / tactic_change / conflict_on_pitch / ignored_pass(REL-05) /
  milestone(首秀/百场) / key_moment 升格标记；
- 每条含：分钟、参与者及角色、判定链摘要、cause_ids（含场外因：
  战术设置 id、关系边、状态值快照）、叙事分（MAT-04）；
- 赛后聚合包：比分、评分表、统计行、key moments、待生成候选清单
  （MOM 表彰/罪人舆论/伤病余波/冲突后续——阶段 7 消费）；
- 文本直播渲染（UX-04）：event 流 + 模板（EVT-06 同管线），支持
  跳过与 2x 速度——比赛展示是可选消费不是强制等待。

**验收标准**：
- [机器] event schema 校验；全部事件 cause 链可回溯到判定链或
  场外因；
- [机器] 赛后候选清单与事件流自洽（无凭空候选）；
- [机器] 关闭直播渲染纯结算模式结果与逐段模式一致（同 seed）。

**评估钩子**：
- 各事件类型每场频率分布（对照现实统计带，进 MAT-06 校准集）。

### MAT-06 结果可信度校准

状态: draft · 优先级: P0 · 依赖: MAT-01, MAT-05, ENG-05

**目的**：守住体育玩家的底线：比分与统计长期分布必须像真的足球。
商业风险表「比赛结果不可信」的机器化防线。

**设计理念**：用现实足球的公开统计形状做校准目标（G2），跑批做
分布检验——可信度不靠手感靠假设检验。

**如何设计**：
- 校准目标带（顶级联赛公开统计的宽容带）：
  场均总进球 2.4-3.0；主胜/平/客胜 ≈ 45/26/29 ±5pp；
  0-0 占比 6-10%；单队 5+ 进球 ≤ 2%；零封率 25-35%；
  红牌 0.15-0.25/场；点球 0.25-0.35/场；
- 强弱响应：实力差 1 个档位的对阵，强队期望胜率 55-65%（爆冷
  空间是叙事刚需，但不能失真）；
- 赛季层面：冠军积分、降级线积分、最大连胜/连败长度落在现实带；
- 跑批检验：每次比赛引擎调参后自动跑 2000 场，KS/卡方检验全指标，
  超带即红灯（ENG-05 基线管线的子集）；
- 校准优先级高于戏剧性：MAT-04/REL-05 等叙事修正的限幅以本卡片
  红线为准——冲突时砍叙事修正。

**验收标准**：
- [机器] 全部校准指标在带内（CI 级测试）；
- [机器] 任何叙事修正开/关对比，指标偏移 < 带宽的 30%；
- [人工] 给 3 名 FM 玩家看 5 个赛季积分榜与比分列表，无人说
  「假」。

**评估钩子**：
- 校准指标仪表板（每次跑批自动更新，红绿灯显示）。

### MAT-07 比赛叙事注入（德比 / 宿敌 / 剧情场）

状态: draft · 优先级: P1 · 依赖: MAT-04, DIR-06, WV-01

**目的**：让背景故事改变一场比赛的「意义参数」而非结果概率：德比、
复仇战、老东家对决、保级生死战自动获得叙事增压。

**设计理念**：注入的是放大器（压力系数、镜头权重、记忆显著度），
不是胜负脚本（MAT-06 红线）。意义是查出来的：从恩怨账本、弧线、
记忆自动识别比赛的故事标签。

**如何设计**：
- 赛前意义扫描：对阵双方查询恩怨账本（DIR-05）、相关弧线（DIR-06）、
  人物记忆（旧主/宿敌/兄弟对决），生成比赛标签集与「赛前看点」
  文本（UX-04 赛前页素材）；
- 标签效果：压力场合系数（MAT-03 的场合项）、key moment 阈值降低
  （MAT-04 镜头更密）、相关人物记忆写入显著度 +20、媒体关注度 ×2
  （CAT-03）；
- 德比固定标签（WV-01 预置恩怨对）+ 动态标签（当季结怨即时生效
  ——上周冲突这周对决，世界反应要快）；
- 赛后意义结算：剧情场的结果写入双方恩怨账本（德比惨案会被记十年，
  DIR-05 慢变量）。

**验收标准**：
- [机器] 标签生成 100% 可回溯到账本/弧线/记忆来源；
- [机器] 剧情场的 key moment 密度高于普通场 ≥ 50%，但比分分布与
  普通场无显著差异（意义≠胜负偏置）；
- [LLM] 「赛前看点」文本抽样「让人想看这场球」≥ 4/5。

**评估钩子**：
- 剧情场占比（目标 15-25%/赛季 [待评估校准]）；剧情场记忆的长期
  引用率 vs 普通场（应显著更高）。
