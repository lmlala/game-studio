# UX — 用户交互主要途径

> 交互哲学：**世界向你汇报，你偶尔出手**。这是 G1（有限控制）的交互
> 表达——玩家大部分时间在读世界、做少量高权重决策，而不是填表格。
> 反面教材即 FM 的表格海与流程轮（用户画像一的痛点）。

### UX-01 信息架构：以叙事流为主轴

状态: draft · 优先级: P0 · 依赖: ENG-04, DIR-07, REL-04

**目的**：定义玩家的「家界面」。信息架构决定玩家感知到的是故事
还是数据库。

**设计理念**：报纸隐喻（ART-01 同构）：主屏 = 今日要闻流，按叙事
重要性排序而非时间排序；数据页是查阅工具，永远不是主路径。

**如何设计**：
- 主屏三栏（报纸版式）：
  头条区——Director 标记的 top1-2 事件（severity×焦点加权）；
  要闻流——notable+ 事件卡片，带「为什么重要」一行（弧线归属/
  涉及核心人物）；
  仪表边栏——三声望（REL-04）、积分位次、下一场倒计时、更衣室
  稳定度，5 个常驻数字以内；
- 事件卡片三动作：展开因果链（UX-06）/ 跳转当事人 / 需要回应的
  带决策入口（UX-02）；
- 焦点驱动排序：spotlight 人物（DIR-07）的事件上浮；
- 信息降噪开关：纪实模式（全量）/ 主编模式（默认，notable+）/
  头条模式（major+，挂机型玩家）。

**验收标准**：
- [机器] 主屏渲染数据全部来自结构化日志查询，无 UI 私有状态（G9）；
- [人工] 新玩家 5 分钟测试：能答出「我的队现在最大的麻烦是什么」
  （世界观验收标准的 UI 承接）；
- [人工] 一周目玩家访谈：主观感受「在追剧」而非「在办公」≥ 60%。

**评估钩子**：
- 各栏目点击分布；数据页 vs 叙事流的停留时长比（验证主轴假设）。

### UX-02 周循环决策面

状态: draft · 优先级: P0 · 依赖: UX-01, EVT-02, CAT-01, MAT-02

**目的**：固化「单周 0-3 个有意义决策」的节奏（MVP 玩法验收第 1 条），
定义全部玩家命令入口。

**设计理念**：决策少而重：每个决策都该让玩家停顿 5 秒以上；可批量
默认的（训练沿用上周）绝不重复要求输入。决策疲劳是经营游戏弃坑
首因之一（PSY-03 关联）。

**如何设计**：
- 周节奏锚点（自动暂停点，可在设置关闭）：
  周一·复盘——上周事件清算 + 需回应事项（0-3 个决策卡）；
  周三·备战——训练重点确认（默认沿用，一键过）+ 人事动作窗口
  （谈话/处罚/表扬/承诺，每周限 2 次防刷，G1 教练精力设定）；
  周五·首发——阵容与战术（MAT-02 五项 + 首发 11 人，默认上轮
  延续 + 助教建议差异高亮）；
  周末·比赛——观看或跳过（UX-04）；
- 决策卡格式统一：情境摘要 + 2-4 个选项 + 每选项的「已知影响 +
  不确定提示」（如「他可能高兴，但队长未必」——预期管理但不
  剧透判定，G1）；
- 全部命令走 Rust 校验（非法命令拒绝并说明，现有 sim 已验收能力
  保持）；
- 自动推进模式：无决策事项时一键到下一锚点；整周挂机产出周报
  （后台运行模式的 UI 面）。

**验收标准**：
- [机器] 玩家命令种类完整覆盖玩法文档可控范围表；非法命令 100%
  被拒并带原因；
- [机器] 跑批模拟玩家：每周需强制输入的决策 ≤ 3，纯确认类 ≤ 2；
- [人工] 一周游戏时长 8-15 分钟（不含比赛观看），节奏自评不拖沓。

**评估钩子**：
- 决策卡选项分布（某选项被选率 > 80% = 假选择，退回设计）；
- 每周实际决策数分布。

### UX-03 人物页与关系图

状态: draft · 优先级: P1 · 依赖: ACT-03, REL-06, UX-01

**目的**：玩家理解「人」的主入口。人物页读起来必须像人物小传，
不是属性面板——情感冷启动的 UI 侧战场。

**设计理念**：把记忆放在比数值更显眼的位置：玩家点开一个球员先看到
「他经历过什么、他怎么看你」，再看到能力条。

**如何设计**：
- 人物页布局（报纸人物专访版式，ART-01）：
  上：形象（ART-02）+ 名字 + 一句话人设（人格特质的自然语言摘要）；
  中：**记忆时间线**（ACT-03 显著度 top8，按时间排列成小传）+
  「对你的态度」（信任档位 + 已知原因列表）；
  下：能力/状态档位条（模糊化，ACT-02 可见性纪律）+ 欲望提示
  （已侦察到的，ACT-04）；
- 关系图页：REL-06 的队级视图（派系着色/张力边/本周剧变高亮），
  点边显示关系小史（变化事件列表）；
- 人物间跳转无死路：任何名字可点（事件卡→人物页→关系边→对方
  人物页——浏览即探索）。

**验收标准**：
- [人工] 给测试者看 3 个人物页各 30 秒，复述「这是个什么人」准确率
  ≥ 70%（与 ACT-01 的 LLM 测试同口径，真人版）；
- [机器] 全部展示数据来自 FFI 查询接口（REL-06 等），Godot 零计算；
- [人工] 关系图页 10 秒读懂测试（REL-06 同条）。

**评估钩子**：
- 人物页访问深度（每会话打开人数）与记忆区滚动率。

### UX-04 比赛日体验

状态: draft · 优先级: P1 · 依赖: MAT-05, MAT-07, EVT-06

**目的**：让比赛成为每周的情绪峰值而非等待进度条，同时尊重「跳过」
权利（结果党与过程党都要服务）。

**设计理念**：文本直播 = 广播剧不是流水账：只播 key moment 与态势
变化，留白让想象工作。比赛页是唯一允许「实时感」的场景。

**如何设计**：
- 赛前页：MAT-07 看点 + 首发对比 + 助教情报（对手状态传闻，带噪声）；
- 直播模式：分钟流（压缩 90 分钟→3-5 分钟真实时间）+ key moment
  插卡（暂停、放大、参与者背景一行）+ 中场/落后时的临场调整窗口
  （换人/心态/喊话，每场 ≤ 4 次干预机会）；
- 快速模式：直接出赛后包（比分/评分/key moments 列表/统计行），
  「值得回看」的 key moment 可展开文本回放；
- 赛后页：更衣室即时反应预览（1-2 条）+ 发布会 3 选 1 口径
  （CAT-03 接口）——比赛的叙事余波当场开始，钩住下一周（PSY-03
  开放回路）。

**验收标准**：
- [机器] 直播/快速两模式同 seed 结果一致（MAT-05 契约）；
- [人工] 直播模式比赛主观时长评价「刚好/想更长」≥ 70%；
- [机器] 临场干预命令延迟生效逻辑（下一控球段）确定且可测。

**评估钩子**：
- 直播 vs 跳过的使用比例及其与玩家留存的相关（过程党/结果党画像）。

### UX-05 战报与分享导出

状态: draft · 优先级: P1 · 依赖: DIR-06, SEED-04, EVT-06

**目的**：把「每个存档是一部足球肥皂剧」变成可带走的实体：赛季战报、
seed 名片、名场面卡。用户画像四（内容创作者）的核心服务，也是副业
项目零预算营销的武器。

**设计理念**：分享物 = 游戏的自来水广告；导出质量按「直接可发社交
媒体」标准设计，不是 txt 转储。

**如何设计**：
- 赛季战报（自动生成，报纸特刊版式）：按弧线分章（DIR-06）+ 固定
  栏目（最大背叛/最佳和解/最荒诞标题/关键比赛/赛季英雄与罪人/
  派系版图变迁/教练三声望曲线）；
- 导出格式：长图（社交媒体）/ Markdown（论坛战报佬）/ 游戏内回顾册
  （多赛季书架，DIR-05 的玩家面）；
- seed 名片与名场面卡：SEED-04 设计的 UI 承接，事件卡片一键生成
  分享图（含 seed+周数坐标）；
- 文案密度控制：整刊 ≤ 1500 字（EVT-06 渲染），宁少勿水。

**验收标准**：
- [机器] 跑批每 seed 战报生成成功率 100%，要素完整；
- [LLM] 战报「陌生人愿意读完」评分 ≥ 3.8/5（EVAL-03 核心指标
  同源）；
- [人工] 验证点 1（市场验证）：战报发 FM 社区，自然互动数据达预设
  阈值——本卡片是全项目第一个对外验证物。

**评估钩子**：
- 导出功能使用率；社区战报转发/回复数（上线后）。

### UX-06 因果追溯交互（「为什么会这样」）

状态: draft · 优先级: P1 · 依赖: ENG-04, EVT-04

**目的**：把 G4 可解释性变成玩家可用的功能：任何事件一键展开因果链。
这是「被骰子惩罚感」的解药（PSY-05），也是本作对 FM 黑箱感的直接
差异化卖点。

**设计理念**：因果链展示是「侦探板」不是「调试器」：用人话+人物头像
+时间线呈现，技术字段全部隐藏。

**如何设计**：
- 事件卡「为什么？」按钮 → 因果板：cause 链 DAG 的人读化渲染
  （每节点 = 头像 + 一句话 + 日期），深度默认 3 层可展开；
- 记忆引用特殊标记：链中出现旧记忆时显示「他还记得 X 周前…」
  （EVT-04 的呼应感可视化）；
- 未知因素的诚实呈现：玩家不可见的暗流（未侦察关系等）显示为
  「？？？」节点——可解释 ≠ 全知（G1 与 G4 的平衡点：解释已发生
  的，保留未发生的）；
- 因果板可分享（UX-05 名场面卡的展开形态）。

**验收标准**：
- [机器] 任意 notable+ 事件因果板生成 < 200ms，零断链；
- [人工] 测试者对 5 个随机事件看完因果板后「理解为什么发生」
  ≥ 80%；
- [人工] 「？？？」节点引发好奇而非恼怒（访谈定性）。

**评估钩子**：
- 因果板打开率（健康带 10-30%：太低=没人需要解释，太高=世界太
  令人困惑）。

### UX-07 上手引导与首小时

状态: draft · 优先级: P2 · 依赖: SEED-02, UX-01, UX-02

**目的**：5 分钟理解处境、30 分钟第一个两难、2 小时记住 3 个人物
——情感冷启动的时间表（世界观验收标准的体验化）。

**设计理念**：教学即剧情：不教 UI 教处境。开局就是一场危机交接
（前任教练为什么走、老板要什么、更衣室谁在等你表态），玩家在
处理第一周事务中自然学会全部交互。

**如何设计**：
- 新手世界偏置：首次游戏强制推荐「中游多事之队」（SEED-02），
  且保证首周有 1 个低风险决策卡 + 首月有 1 个真两难（戏剧种子的
  教学化注入）；
- 渐进披露：关系图/派系/声望页在首次相关事件后解锁并头条提示
  （「你的助教觉得有必要给你看看更衣室的小团体」——解锁也是叙事）；
- 助教 = 教学代言人：所有玩法提示以助教留言形式出现（ACT-07 的
  信息过滤器角色兼任导师，零第四面墙打破）；
- 首小时埋点：完成首周/首赛/首决策卡/首次打开因果板的漏斗。

**验收标准**：
- [人工] 5 名新测试者首小时无卡死、漏斗完成率 ≥ 80%；
- [人工] 2 小时后测试者能不看屏幕说出 3 名自己队球员的名字和
  「他们是什么人」（情感冷启动的直接检验，全项目最重要的真人
  验收）。

**评估钩子**：
- 首小时漏斗数据；首周目弃档点分布。
