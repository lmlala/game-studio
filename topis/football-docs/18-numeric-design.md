# NUM — 数值设计原则与方法

> 全库散落的数值（各卡片的初始值与 [待评估校准] 标记）的**总宪法**：
> 量纲、堆叠规则、随机预算、速率表、频率配额、校准方法。
> 标注 **[锁死]** 的条目是全项目硬约定，修改视同改架构，需人工签收；
> 其余为初始值，按 NUM-06 方法校准。冲突时本文件优先于各系统卡片。

### NUM-01 全局量纲与标度 [锁死]

状态: draft · 优先级: P0 · 依赖: 无

**目的**：统一全部数值的量纲，杜绝"这个 80 和那个 80 不是一回事"——
多人协作与 agent 批量产卡时量纲漂移是数值系统腐烂的第一原因。

**设计理念**：标度种类越少越好；每种标度绑定唯一语义和唯一展示规则。

**如何设计**：
- [锁死] 标度全集（任何卡片不得发明新标度）：
  | 标度 | 范围 | 用途 | 展示规则 |
  | --- | --- | --- | --- |
  | 能力/状态 | 0-100 整数 | 技能、心情、压力、自信、体能、声望 | 玩家只见 5 档文字（G1），调试 UI 可见数值 |
  | 关系 | -100~+100 整数 | 五维关系、记忆极性 | 玩家见档位+原因列表 |
  | 比例 | 0.0-1.0 | 概率、占比、执行度 | 玩家见自然语言档（很可能/可能/说不准） |
  | 修正器 | 乘数 0.5-2.0 或 加数 ±15 | 事件权重、判定修正 | 永不直接展示 |
  | 荒诞度 | 0-3 枚举 | 事件分级 | 不展示数值，体现在文风 |
  | 评分 | 1.0-10.0 一位小数 | 比赛个人评分 | 直接展示（足球惯例） |
  | 严重度 | 5 级枚举 | 日志/事件分级 | 映射字号与图标（ART-03） |
- [锁死] 内部计算允许浮点，落盘与展示前量化到标度精度；
- [锁死] 隐藏状态（人格、欲望强度、未暴露关系）永不向玩家展示数值，
  只通过行为、助教报告（带噪声）与事件体现（G1）；
- 5 档文字映射 [锁死]：0-19 糟糕 / 20-39 低迷 / 40-59 一般 /
  60-79 不错 / 80-100 出色（关系档位同构：死敌/紧张/平淡/友好/亲密）。

**验收标准**：
- [机器] 全库卡片扫描：出现数值范围声明的地方均符合标度全集（agent
  门禁可加正则规则）；
- [机器] 代码层：标度类型用 newtype 封装（Rust），跨标度运算编译失败；
- [机器] UI 层无任何隐藏状态的原始数值泄漏（接口快照测试）。

**评估钩子**：
- 跑批中各标度变量的分布快照（越界值数 = 0 为硬指标）。

### NUM-02 修正器堆叠规则 [锁死]

状态: draft · 优先级: P0 · 依赖: NUM-01

**目的**：规定多个修正器同时作用时的计算顺序与上限——堆叠规则不统一，
平衡性就无法推理，玩家也无法形成可学习的直觉。

**设计理念**：加法在类内、乘法在类间、全局有夹钳；任何修正器不能
反转基础能力的相对强弱（弱者吃满 buff 不应稳赢强者，G2 可信度）。

**如何设计**：
- [锁死] 判定值流水线（所有引擎判定统一走此管线，ENG-03 配套）：
  `最终值 = clamp(基础值 × Π(类间乘数) + Σ(类内加数), 下限, 上限)`
  1. 类内（同来源类别的多个修正先**加法**合并，再夹到类别带宽）；
  2. 类间（人格/状态/关系/战术/场合 五类之间**乘法**）；
  3. 全局夹钳。
- [锁死] 五类修正的带宽（对比赛判定）：
  人格 ±10% · 状态 ±15% · 关系 ±8% · 战术 ±10% · 场合 ±10%；
  全局夹钳：合成修正 ∈ [0.7, 1.3]——基础能力始终是主项；
- [锁死] 符号保护：任何修正链不得把概率压到 0 或抬到 1（夹在
  [0.02, 0.98]），"几乎不可能"也要留奇迹缝隙（叙事需要）；
- [锁死] 应用顺序确定：类别按固定枚举序计算（G5）；
- 非比赛域（事件权重、关系变化量）同构管线，带宽各域卡片自定但
  不得超过 ×3 / ÷3 总幅（防权重爆炸）。

**验收标准**：
- [机器] 流水线单测：构造满 buff 弱者 vs 零 buff 强者（基础差 ≥ 20），
  弱者胜率 < 40%；
- [机器] 全部判定的合成修正分布在夹钳内（跑批断言）；
- [机器] 修正器注册表：每个修正器声明类别与带宽，越带宽编译/加载失败。

**评估钩子**：
- 修正器贡献度分解报告（哪类修正长期主导 = 设计失衡信号）。

### NUM-03 随机性预算

状态: draft · 优先级: P0 · 依赖: NUM-02, SEED-01

**目的**：规定"哪里允许掷骰子、掷多大"。随机是叙事的燃料也是
公平感的毒药（PSY-05），必须定量管理。

**设计理念**：随机只出现在**判定**环节（结果掷骰），不出现在**状态**
环节（状态变化必须有 cause）；方差是人格的表达维度而不是全局常数。

**如何设计**：
- [锁死] 随机三原则：
  1. 状态变化零随机：任何状态 delta 都由事件/判定结果驱动（G3/G4）；
  2. 判定随机有界：判定噪声 σ ≤ 判定值的 12%（初始值，[待评估校准]）；
  3. 方差人格化：个体噪声 σ × (1.5 − 情绪稳定/100)——稳定老将
     σ≈0.5 倍，玻璃心新星 ≈1.4 倍（ACT-02/MAT-03 的数值落点）；
- 预期违背率目标：助教预测 vs 实际的偏差率 10-20%（PSY-02 锚定），
  作为全局随机量是否健康的代理指标；
- 厄运保险丝：同一 Actor 连续 3 次低概率坏结果后，第 4 次判定
  噪声向均值收缩 50%（伪随机保底，玩家感知的"公平"比数学公平
  重要；不改变期望只压尾部）[待评估校准——可能与硬核玩家口味
  冲突，A/B 后定]；
- 全部噪声走对应域 RNG 流（SEED-01），禁止全局共享流。

**验收标准**：
- [机器] 状态变化日志 100% 带 cause_id（与 ENG-04 同口径，复引）；
- [机器] 判定噪声实测 σ 在带内；方差与情绪稳定的相关性显著为负；
- [机器] 保险丝触发率 < 5% 且触发记录可审计。

**评估钩子**：
- 预期违背率实测；"被骰子惩罚"类内测反馈计数（PSY-05 共用）。

### NUM-04 变化速率与半衰期总表

状态: draft · 优先级: P0 · 依赖: NUM-01

**目的**：把散落各卡的时间常数收编成单一权威表——快中慢三层时间
尺度（DIR-05 理念）的数值落点。各系统卡片**引用本表，不再自带数字**。

**设计理念**：速率决定游戏的"呼吸频率"；同层变量速率必须同量级，
否则玩家无法建立时间直觉。

**如何设计**：
- 速率总表（初始值，全部 [待评估校准]，但**层级归属 [锁死]**）：
  | 变量 | 层 | 速率/半衰期 | 来源卡 |
  | --- | --- | --- | --- |
  | 体能 | 快 | 比赛 -25±10 / 休息日 +15 | CAT-02 |
  | 心情 | 快 | 半衰期 5 日（情绪稳定调制 3-7） | ACT-02 |
  | 自信 | 快中 | 半衰期 14 日 | ACT-02 |
  | 压力 | 中 | 积分器，泄压靠事件 | ACT-02 |
  | 关系五维 | 中 | 自然回归 ±1/周 向初始倾向 | REL-01 |
  | 派系凝聚 | 中 | 滞回 2 周成/3 周散 | REL-03 |
  | 记忆显著度 | 慢 | -5/月，引用 +15 | ACT-03 |
  | 三声望 | 慢 | 半衰期 1 赛季 | REL-04 |
  | 俱乐部国力 | 极慢 | 赛季结算更新 | DIR-05 |
  | 荒诞预算 | 中 | +1.0/周 上限 4.0 | DIR-04 |
- [锁死] 跨层禁令：快变量不得直接改慢变量（心情不能直接改声望），
  必须经事件中介——保证慢变量的每次变化都有故事；
- 表的维护规则：新增时间常数必须入本表并标层级；agent 修订各系统
  卡片时数值与本表冲突视为门禁错误（后续接入 agent 规则）。

**验收标准**：
- [机器] 跑批中各层变量的自相关时间与层级定义吻合（快 < 2 周 <
  中 < 2 月 < 慢）；
- [机器] 慢变量变化 100% 带事件 cause（跨层禁令的执行验证）；
- [人工] 本表与各系统卡片数值无冲突（agent 一致性巡检项）。

**评估钩子**：
- 各变量自相关时间的版本间漂移监控。

### NUM-05 戏剧经济总配额（频率预算）

状态: draft · 优先级: P0 · 依赖: NUM-04, DIR-02

**目的**：把"多久发生一次"集中成一张戏剧经济表——事件频率是体验
节奏的硬约束，分散在各卡会互相叠加成事件洪水或荒漠。

**设计理念**：以"玩家每周注意力 = 处理 0-3 件事"反推全部配额
（UX-02 锚点）；配额是预算不是目标——Director 在预算内自由发挥。

**如何设计**：
- 周配额（玩家队视角，初始值 [待评估校准]，**总量上限 [锁死]**）：
  | 类别 | 每周带 | 上限来源 |
  | --- | --- | --- |
  | notable+ 事件总量 | 3-8 件 | ENG-04 写入预算 ≤12 的子集 |
  | 需玩家决策 | 0-3 件 | UX-02 [锁死] |
  | major+ 事件 | 0-2 件 | 注意力稀缺 |
  | 社交 notable | ≤5 件 | REL-02 频控 |
- 赛季配额（每队）：老板干预 1-4 · 球迷组织行动 1-3 · 协会动作
  全联赛 1-2 · 重伤 1-3 人次 · 轻伤 8-20 · 派系 1-3 个 ·
  absurdity-2 事件 3-6 件 · season_defining 1-3 件；
- [锁死] absurdity 分布带：0 级 55-65% / 1 级 25-35% / 2 级 8-12%
  （WV-05 的数值定稿）；
- 配额执行机制：Director 选牌时按滑动窗口核算（DIR-01 输入），
  超配类别候选分 ×0.3 软压制而非硬禁（留给极端剧情破例空间）；
- 全部配额项自动映射 15-mentor 验收型规则（`Count ... Between`）。

**验收标准**：
- [机器] 100 seed 跑批：各配额项 P10-P90 落在带内；
- [机器] 配额表 → 漏斗规则的生成器存在且双向同步（单一真值源）；
- [人工] 内测者主观节奏评价与配额带修订记录联动（每次改带写理由）。

**评估钩子**：
- 配额项实测分布仪表板（每次跑批刷新，超带红灯）。

### NUM-06 校准方法论与参数三色制

状态: draft · 优先级: P0 · 依赖: NUM-05, EVAL-04

**目的**：定义数值从拍脑袋到收敛的工作流，以及 [待评估校准] 标记的
生命周期——防止"永远待校准"。

**设计理念**：参数分三色管理，校准是实验不是手感（EVAL-04 方法的
数值侧落地）。

**如何设计**：
- 参数三色制 [锁死]：
  - **红（锁死）**：量纲、堆叠规则、跨层禁令、配额总量上限——改动
    = 改架构，人工签收 + changelog；
  - **黄（校准中）**：带 [待评估校准] 的初始值——必须同时声明
    «目标指标 + 可接受带»，否则不准入卡（agent 门禁项）；
  - **绿（已收敛）**：经 EVAL-04 配对实验确认落带的值——移除标记，
    记录实验 id；
- 黄色参数生命周期 [锁死]：入卡 → 进入校准队列 → 单变量扫参
  （SEED-05 网格）→ 配对对比（同 seed 集）→ 落带转绿 / 两轮不收敛
  升级人工重新设计；每个里程碑结束时黄色参数存量必须下降；
- 灵敏度优先：先做全参数粗灵敏度扫描（每参数 3 点），高灵敏参数
  优先精校，低灵敏参数直接转绿（不值得精调）；
- 所有参数外置配置文件（ENG-03），扫参不改代码不重编译。

**验收标准**：
- [机器] 参数注册表存在：每参数带颜色、目标指标、当前实验 id；
- [机器] 黄色参数无目标指标即配置加载失败；
- [人工] 每里程碑黄色参数清单复盘（数量趋势必须下降）。

**评估钩子**：
- 黄色参数存量曲线；每参数的灵敏度系数表（EVAL-04 产出）。

### NUM-07 数值红线（反膨胀与反退化）[锁死]

状态: draft · 优先级: P1 · 依赖: NUM-01, NUM-02

**目的**：列举数值系统的长期死法并预埋禁令——数值膨胀和退化都是
慢性病，等看见症状已经晚了。

**设计理念**：单机叙事游戏不需要数值成长爽（不是 RPG），需要的是
**差异稳定可读**；一切让数字变大的诱惑都要过本卡审判。

**如何设计**：
- [锁死] 反膨胀：联赛能力总池逐赛季漂移 ≤ ±2%（新老交替守恒，
  ACT-06 配套）；无全局成长倍率；无"下赛季全员+5"式通胀；
- [锁死] 反复利失控：修正器不得引用另一个修正器的输出（链长 = 1），
  复合效果只能通过事件中介跨 tick 实现；
- [锁死] 反伪精度：展示层禁止两位小数（评分一位小数除外）；设计
  文档中的数值精度不得超过校准实验能分辨的精度；
- [锁死] 反隐性难度：不做随玩家战绩缩放的橡皮筋数值（AI 不偷看
  玩家状态加 buff）——涌现叙事的可信度高于挫折保护，挫折保护走
  叙事手段（DIR-08）不走数值手段；
- 守门测试：10 赛季连跑，联赛均值/方差/冠军积分无单调漂移。

**验收标准**：
- [机器] 10 赛季跑批：能力总池、评分均值、进球率的线性趋势斜率
  与零无显著差异；
- [机器] 修正器注册表静态检查：无修正器间引用；
- [机器] 橡皮筋禁令：引擎代码无玩家身份分支的数值路径（lint）。

**评估钩子**：
- 长跑批的标度分布漂移报告（每版本对比）。
