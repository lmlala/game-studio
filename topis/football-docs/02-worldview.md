# WV — 世界观设计

> 继承 `docs/02-world/setting-and-rules.md` 的设定基调，按卡片协议重组并细化。
> 核心立场：G2（足球规则不架空，足球世界架空）。

### WV-01 架空国家与地区性格

状态: draft · 优先级: P1 · 依赖: 无

**目的**：给所有俱乐部、人物、事件一个有差异的社会土壤，使「同一种事件
发生在不同地区有不同走向」，提高事件复用度（G10）。

**设计理念**：地区不是地图装饰，是事件权重修正器。瓦洛尼亚联盟设定保留：
一个把足球当社会情绪出口的经济停滞小国。

**如何设计**：
- 1 个国家，5 个地区标签：港口区 / 首都圈 / 北部矿带 / 南部大学城 /
  边境贸易区（沿用现有设定）；
- 每个地区是一组数值修正：球迷易怒度、媒体密度、青训倾向、外援比例、
  身体对抗偏好、地方主义强度（各 0-100，初始值入配置文件）；
- 地区修正作用点：事件权重（EVT-02）、球迷引擎参数（CAT-05）、媒体引擎
  语气（CAT-03）、世界生成的人物特质分布（SEED-02）；
- 地区之间预置 2-3 对历史恩怨（德比素材，MAT-07）。

**验收标准**：
- [机器] 每支球队恰好挂 1 个地区标签；地区参数文件 schema 校验通过；
- [机器] 跑批中同一事件类型在不同地区的触发频率差异 ≥ 20%（证明修正器生效）；
- [LLM] 抽取两支不同地区球队的赛季战报，评估器能盲猜出其地区性格（≥ 3/5）。

**评估钩子**：
- 按地区分组的事件类型分布熵；地区间德比事件触发率。

### WV-02 联赛结构与赛季框架

状态: draft · 优先级: P0 · 依赖: WV-01

**目的**：定义模拟的时间骨架。所有叙事节奏（DIR-02）最终锚定在赛程上。

**设计理念**：G2、G8。MVP 一个顶级联赛双循环；杯赛后置但**赛季框架预留
插槽**，避免日后伤筋动骨。

**如何设计**：
- 8-12 队双循环（14-22 轮），每周一赛，赛季约 5-6 个月模拟时间；
- 赛季阶段标签（供 Director 与事件前置条件引用）：`preseason / opening /
  mid_crisis_window / transfer_window / run_in / finale / offseason`；
- 积分规则现实标准：胜 3 平 1 负 0，同分比净胜球→进球→交锋；
- 赛季目标系统：每队开局由老板引擎（CAT-04）生成目标（夺冠/欧战位/
  中游/保级），目标与实际排名的差值是全局压力源（DIR-02 输入）；
- 升降级 MVP 用「影子处理」：末位球队触发降级叙事事件与人员动荡，但
  下赛季联赛成员不变 [待评估校准，正式版换真实升降级]。

**验收标准**：
- [机器] 任意 seed 生成的赛程满足：双循环完整、主客平衡、无同队同轮冲突；
- [机器] 赛季阶段标签在日志中正确标注，跑批可按阶段聚合事件；
- [机器] 赛季结束产出结算记录：冠军、降级、射手榜、目标达成表。

**评估钩子**：
- 各赛季阶段的 major 事件密度（应呈 opening 低 → mid 高 → finale 最高）。

### WV-03 俱乐部文化标签

状态: draft · 优先级: P1 · 依赖: WV-01

**目的**：让 8-12 支球队各自「像一个有历史的俱乐部」而非换色队徽，
并为更衣室默认价值观提供初始参数。

**设计理念**：文化标签 = 事件池过滤器 + 初始关系倾向 + 球迷/老板预期模板。

**如何设计**：
- 标签池（每队 2-3 个）：老牌豪门 / 工人球队 / 财团新贵 / 青训圣地 /
  保级专业户 / 港口德比队 / 大学城球队（沿用现有 7 个，允许扩到 10）；
- 每个标签定义四元组：事件权重修正表、球迷容错曲线参数、老板模板偏好、
  更衣室初始价值观向量（忠诚/纪律/进攻美学/本地认同，各 0-100）；
- 标签冲突表：互斥对（如 老牌豪门×财团新贵）由世界生成器规避（SEED-02）；
- 标签可演化：连续 3 赛季满足条件可获得/失去标签（世界线演化 DIR-05 的
  素材，如保级专业户夺冠后转型）。

**验收标准**：
- [机器] 生成的联赛中标签组合无互斥违例，且 7 个标签每个至少出现一次；
- [LLM] 给评估器两队的事件流（隐藏队名标签），判别其文化标签准确率 ≥ 60%；
- [人工] 每个标签至少绑定 5 个专属事件定义（EVT-07）。

**评估钩子**：
- 按标签分组的事件类型分布差异度；标签演化事件的多赛季触发率。

### WV-04 主要势力与社会角色

状态: draft · 优先级: P0 · 依赖: WV-03

**目的**：定义更衣室之外的戏剧引擎燃料：老板、媒体、球迷组织、协会、
家人朋友。G1 的「不可控部分」主要由它们承载。

**设计理念**：每个势力都有自己的利益函数，没有人「为了球队」——黑色幽默
来自利益错位（G6 的因果锚点库）。

**如何设计**：
- 老板：性格模板（成绩急躁型/资产运作型/明星收藏型/政治攀附型/吉祥物
  迷信型），各带干预倾向与满意度函数（CAT-04）；
- 媒体：3-6 名记者 Actor，各有偏见对象、消息源（更衣室内线）、流量压力；
  标题生成走结构化事件→语气模板（CAT-03）；
- 球迷组织：每队 1-2 个（老派看台/青训保护派/反老板派/成绩极端派/球员
  个人崇拜派），有情绪值与组织行为（横幅、抗议、TIFO）（CAT-05）；
- 协会：低频荒诞制度发生器（临时规则、纪律处罚、形象工程），每赛季
  1-2 次重大动作 [待评估校准]；
- 家人朋友：不做完整 Actor，做「影响标签」挂在球员身上（家庭压力/
  狐朋狗友/模范家庭），影响状态引擎与事件前置（ACT-07）。

**验收标准**：
- [机器] 每个势力至少有 1 个引擎消费其状态、3 种事件以其为主语（EVT-07）；
- [机器] 跑批中老板干预、媒体风暴、球迷行动每赛季每队触发次数落在
  设计区间（各 1-4 次 [待评估校准]）；
- [LLM] 评估器对「势力行为是否符合其利益函数」打分 ≥ 4/5。

**评估钩子**：
- 各势力引发的事件占比（任何势力长期 < 5% 即为设计死区）。

### WV-05 荒诞边界与内容红线

状态: draft · 优先级: P0 · 依赖: 无

**目的**：把「荒诞但不胡闹」写成可执行规范，给 DIR-04 荒诞调制器和
内容写作（EVT-07）共同遵守的边界。

**设计理念**：G6。荒诞是现实逻辑在高压下的夸张延伸，不是随机搞笑。

**如何设计**：
- 可以写：职业压力、利益冲突、派系斗争、媒体夸张、老板荒唐、协会官僚、
  家庭压力、经纪人煽动、球迷文化、德比仇恨、受伤背叛和解退役重生；
- 谨慎写（需要 absurdity ≤ 1 且有强因果）：现实政治映射、影射真实俱乐部；
- 禁止写：真实授权名、可识别真实人物影射、平台违规仇恨表达、把严肃创伤
  当随机笑料；
- 荒诞分级样例库（供 LLM 评估对齐用，每级 ≥ 10 例）：
  - 0 现实：主力伤停、续约谈崩；
  - 1 夸张：记者把轮休写成"清洗老臣"；
  - 2 荒诞：老板梦见吉祥物哭泣要求换队长；球迷因门将胡子分裂两派；
  - 3 超现实：**禁止**（无因果锚点的魔幻事件）。

**验收标准**：
- [机器] 事件定义表中不存在 absurdity=3 的事件；
- [机器] absurdity≥2 的事件 100% 带 ≥ 1 个因果锚点字段（性格/利益/压力/
  误会/记忆/环境之一）；
- [LLM] 抽样荒诞事件做「荒诞但可信」评分，均值 ≥ 3.8/5（EVAL-03 指标）。

**评估钩子**：
- absurdity 分布（目标：0 级 ~60% / 1 级 ~30% / 2 级 ~10% [待评估校准]）。

### WV-06 架空命名与本地化

状态: draft · 优先级: P2 · 依赖: WV-01

**目的**：批量生成不撞车、不侵权、带地区风味、中英双语可用的人名/队名/
地名/媒体名。

**设计理念**：名字是玩家对架空人物建立情感的第一接触点（对应"情感冷
启动"风险），生成质量直接影响代入感。

**如何设计**：
- 音节库方案：每个地区一组姓/名音节库 + 组合规则，生成后查重；
- 队名 = 城市名 + 类型词（竞技/联/城/工人/大学），与文化标签联动
  （工人球队禁用"皇家"类前缀）；
- 中英双语：生成器同时产出 zh/en 形式（音译表驱动），UI 仅 en/zh（项目
  i18n 约束）；
- 黑名单：现实知名俱乐部/球员名的编辑距离 ≤ 2 的候选全部丢弃。

**验收标准**：
- [机器] 生成 10000 人名零重复率 ≥ 99%；黑名单过滤测试通过；
- [人工] 抽 20 个队名读感测试：能猜出地区/文化的 ≥ 12 个。

**评估钩子**：
- 战报 LLM 评估附加题：「名字是否破坏沉浸感」（是/否计数）。
