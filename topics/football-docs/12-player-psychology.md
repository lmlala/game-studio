# PSY — 游戏心理学与粘性设计

> 把「为什么玩家会留下来」从直觉变成可设计、可验证的机制。
> 基础玩家分类沿用产品文档四画像：FM 叙事疲劳者 / 系统叙事玩家 /
> 轻度足球经营者 / 内容创作者。

### PSY-01 玩家动机映射

状态: draft · 优先级: P1 · 依赖: 无

**目的**：把四类画像翻译成动机结构与设计供给清单，使每张设计卡都能
回答「这服务于谁的哪种动机」。

**设计理念**：不为不存在的玩家设计。四画像的主导动机各不相同，
但共享一个核心：**对涌现故事的叙事掌控感**（我影响了它，但它不
属于我）。

**如何设计**：
- 动机映射表：
  FM 疲劳者——胜任感（决策有杠杆）+ 新奇（事件不模板化）→ 供给：
  战术真实杠杆（MAT-02）、事件反疲劳（EVT-05）；
  系统叙事玩家——理解系统的求知欲 + 失败审美 → 供给：因果板
  （UX-06）、失败叙事（DIR-08）、派系涌现（REL-03）；
  轻度足球玩家——低门槛代入 + 情绪过山车 → 供给：周决策 ≤ 3
  （UX-02）、比赛情绪峰（UX-04）；
  创作者——可讲述素材 + 身份展示 → 供给：战报/seed 名片（UX-05/
  SEED-04）；
- 每张新设计卡的「目的」字段建议注明主要服务画像（设计 agent 的
  门禁检查项，见 `apps/design-studio/docs/`）；
- 冲突仲裁原则：动机冲突时优先序 = 系统叙事玩家 > FM 疲劳者 >
  创作者 > 轻度玩家（核心受众的交集判断，首发策略）。

**验收标准**：
- [机器] 全卡片库扫描：四画像各被 ≥ 5 张卡片显式服务；
- [人工] 内测问卷按画像分组，各组「这游戏为我做的」自由作答与
  设计供给清单匹配度定性评审。

**评估钩子**：
- 内测分画像留存差异（某画像断崖 = 供给清单失效）。

### PSY-02 核心情绪回路

状态: draft · 优先级: P0 · 依赖: DIR-02, UX-02, UX-04

**目的**：定义玩家单次会话的情绪曲线设计目标——粘度的微观基础是
「这一小时的感受」，不是宏观功能清单。

**设计理念**：本作的情绪回路是「焦虑—决断—揭晓—余波」四拍，
以周为单元、比赛为峰值。区别于建造类的「积累满足」，这是剧情向
的「悬念满足」——设计要保护悬念而不是消灭悬念（G1 的心理学版）。

**如何设计**：
- 四拍节奏与 UX-02 锚点对齐：周一余波（上周后果揭晓，情绪结算）
  → 周三决断（人事/训练，掌控感注入）→ 周五焦虑（首发抉择，
  最大不确定性时刻）→ 周末揭晓（比赛，峰值）；
- 焦虑的剂量控制：同时活跃的「悬而未决」事项 2-4 件（弧线进行中、
  承诺待兑现、伤员归期）；< 2 注入新钩子（Director 责任，DIR-02
  泄压的反向），> 4 强制收束（DIR-06 超期机制）；
- 揭晓的偶尔反转：助教情报噪声（ACT-07）保证 10-20% 的预期违背率
  ——全可预测无悬念，全不可预测无掌控；
- 余波的确认感：每个重大决策在 1-3 周内必有可见回响（哪怕是
  「暂无风波」的明确反馈）——决策不能石沉大海（G3 的体验面）。

**验收标准**：
- [机器] 跑批模拟玩家视角：悬而未决事项数的时间序列在 2-4 带内
  占比 ≥ 70%；
- [机器] 重大决策的 3 周内回响覆盖率 = 100%；
- [人工] 会话结束访谈「现在停下来难受吗（有牵挂吗）」≥ 60% 有
  牵挂——见 PSY-03。

**评估钩子**：
- 预期违背率实测；会话时长分布（健康带 30-90 分钟，过长警惕
  疲劳透支）。

### PSY-03 留存钩子：开放回路设计

状态: draft · 优先级: P0 · 依赖: PSY-02, DIR-06, UX-04

**目的**：让每次退出游戏时玩家心里留着 1-2 个「我想知道后来怎样」
——次日回访的心理机制（蔡格尼克效应的叙事应用）。

**设计理念**：用未完成的故事而非未领取的奖励做留存（拒绝体力值/
签到式黑模式——买断制单机的信用，也是 Steam 评价区的雷区）。

**如何设计**：
- 会话末状态检查：玩家点击退出/存档时，若无活跃悬念（PSY-02 计数
  = 0），Director 在退出前最后一个 tick 允许提前发一张「钩子牌」
  （低成本风闻类：转会传闻/更衣室风声/记者预告）[待评估校准——
  此机制有操纵感风险，A/B 验证后决定保留]；
- 赛程自带钩子：退出画面显示下一场对阵 + 一句看点（MAT-07 赛前
  扫描的复用）；
- 多赛季钩子：赛季末战报最后一页 = 下赛季悬念预告（转会窗风险
  名单/老板野心变化/到期合同），翻完战报就想开新赛季；
- 红线：钩子全部来自真实模拟状态，禁止虚假预告（诚实性 > 留存，
  G4 的信用延伸）。

**验收标准**：
- [机器] 钩子内容 100% 可回溯到真实世界状态（无凭空预告）；
- [人工] 内测 7 日回访率作为北极星观测（不设硬指标，记录基线）；
- [人工] 「感到被游戏操纵」负反馈率 < 5%（钩子牌机制的否决条件）。

**评估钩子**：
- 退出时活跃悬念数分布；次会话首动作（直奔悬念事项 = 钩子生效）。

### PSY-04 失败心理与挫折管理

状态: draft · 优先级: P0 · 依赖: DIR-08, UX-06

**目的**：让失败被体验为「故事转折」而非「惩罚结算」，保住失败档
的继续游玩率——涌现叙事游戏的留存命门（RimWorld 的 losing is fun
能否移植到胜负题材，是本作最大的心理学赌注）。

**设计理念**：体育题材的失败天然比殖民地全灭温和（输球≠存档毁灭），
这是题材优势；危险在「连败螺旋的无力感」。对策不是降难度，是保证
失败期的**代理感**（仍有有意义的事可做）与**意义感**（失败在产出
故事）。

**如何设计**：
- 连败期的决策供给：DIR-08 已定失败期人际事件配额；UX 侧保证连败
  周的决策卡 ≥ 平时（危机=更多人来找你，而非更少）；
- 失败归因的可见性：连败复盘页（因果板的赛季视图）让玩家看到
  「输在哪」——可归因的失败激发再试，不可归因的失败导致习得性
  无助（UX-06 的存在理由之一）；
- 解雇的软着陆：下课触发「告别结算」——更衣室告别反应（谁惋惜/
  谁庆祝，忠诚的审判时刻）+ 生涯战报存册 + 赋闲期新开局（DIR-08）；
  解雇是章节结束不是 game over 画面；
- 难度表达：不设难度选项，用球队选择承担难度（保级队=困难模式），
  推荐页明示叙事密度而非难度星级（失败多=故事多的价值观前置）。

**验收标准**：
- [机器] 连败期（3+）的每周决策卡数 ≥ 全季均值；
- [人工] 内测被解雇玩家的继续游玩率 ≥ 50%（核心赌注的直接检验）；
- [LLM] 解雇告别结算文本「让人想截图」≥ 4/5。

**评估钩子**：
- 弃档点与战绩的相关性（连败第几场弃档最多 = 无力感阈值定位）。

### PSY-05 公平感与可解释性信任

状态: draft · 优先级: P1 · 依赖: UX-06, ENG-04, DIR-01

**目的**：维护玩家对系统的根本信任：「这个世界讲理，虽然不可控」。
信任崩塌的形态 = Steam 差评「纯随机骗钱模拟器」。

**设计理念**：人对随机的容忍度取决于事后可解释性与事前预警密度，
而不是实际概率。设计投资应放在解释与预警，而非降低随机本身
（G1 要求保留随机）。

**如何设计**：
- 重大负面事件的预警纪律：解雇（CAT-04 已定 ≥ 2 次预警）、核心
  球员转会要求、派系反弹等 major 负面事件，必须存在玩家可见的
  前兆链（事后因果板可指认「其实早有信号」）；
- 概率诚实呈现：决策卡的不确定提示用自然语言档位（很可能/可能/
  说不准）且长期校准（说「很可能」的事 70%+ 要发生——文案概率
  审计）；
- 坏运气的戏剧补偿：纯随机厄运（伤病潮等）触发 Director 的叙事
  关照（DIR-08 转向人际后果牌）——机制上不补偿数值，叙事上给
  足镜头（「悲剧也要拍得好看」）；
- 玩家责任的温和归因：复盘页呈现因果但不打分不审判——玩家自己
  得出「是我害了他」比系统宣判强一万倍（叙事掌控感纪律）。

**验收标准**：
- [机器] major 负面事件的前兆链覆盖率 = 100%；
- [机器] 不确定文案的概率校准审计：各档位实际发生率在承诺带内；
- [人工] 内测问卷「这游戏耍赖吗」否定率 ≥ 85%。

**评估钩子**：
- 因果板在负面事件后的打开率与后续会话留存的相关性（解释是否
  真的修复信任）。

### PSY-06 长线粘性：生涯与世界的复利

状态: draft · 优先级: P1 · 依赖: DIR-05, ACT-06, UX-05

**目的**：30 小时后还有什么？答案必须是「我和这个世界的共同历史」
——多赛季粘性来自积累的不可替代性。

**设计理念**：单赛季靠悬念（PSY-03），多赛季靠**沉淀**：你提拔的
青训成了队长、你卖掉的人回来复仇、你的执教风格被媒体起了绰号。
世界记得你（DIR-05 账本），所以这个存档无法被新档替代。

**如何设计**：
- 教练生涯档案：执教履历、信任过/牺牲过的人员名单（高显著度记忆
  的玩家侧聚合）、媒体给玩家的标签演化（CAT-03 长期叙事）、
  生涯关系网（恩人/宿敌/徒弟）；
- 复利事件池：跨赛季专属事件（旧部重逢/复仇战/师徒对决/十年
  德比账），触发条件依赖账本深度——玩得越久事件池实际越大
  （对抗内容消耗的结构设计，DIR-05/EVT-07 联动）；
- 世代感知：青训球员以「你执教第 N 年入队」标记，其成长弧自动
  关联玩家决策史（「他还记得你当年没放弃他」类记忆引用）；
- 成就系统的叙事化：不做「赢 100 场」，做「邪教名帅」「更衣室
  屠夫」「青训教父」等风格认证（执教行为统计的命名仪式，可
  分享，UX-05）。

**验收标准**：
- [机器] 3 赛季档的跨赛季引用事件 ≥ 5 次/季（复利在发生）；
- [机器] 生涯档案数据完整性（全部来自日志聚合，零手工状态）；
- [人工] 内测 3+ 赛季玩家访谈「换新档可惜吗」≥ 70% 可惜。

**评估钩子**：
- 档案平均赛季数分布；跨赛季事件的玩家面曝光率。

### PSY-07 健康设计红线

状态: draft · 优先级: P2 · 依赖: PSY-03

**目的**：明确不做哪些粘性手段——买断制叙事游戏的口碑资产保护。

**设计理念**：粘性来自「想知道后来」，不来自「怕损失进度」。
所有利用焦虑而非好奇的机制都是负资产。

**如何设计**：
- 禁止清单：每日签到 / 限时事件 FOMO / 体力恢复等待 / 进度衰减
  惩罚（离线时世界不偷偷运行）/ 虚假紧迫倒计时 / 操纵性弹窗
  （「真的要退出吗？你的球员需要你」类文案）；
- 允许清单的边界：PSY-03 钩子牌必须是真实模拟产物；退出永远
  一键完成且自动存档；
- 会话时长尊重：90 分钟连续游玩后，下一个周一锚点附加温和的
  章节感提示（「赛程过半，是个存档的好地方」）——帮助停下而非
  阻止停下。

**验收标准**：
- [机器] 禁止清单逐项代码审计通过；
- [人工] Steam 评价区零「黑模式」类差评（上线后持续监测）。

**评估钩子**：
- 会话时长分布的长尾监控（极端长会话占比异常上升时复查设计）。
